## How to Use (Agent Onboarding)

**Interaction pattern:** `CLI (Bash) + embedded agent instructions`

```bash
# Install (pick one)
cargo install cx-cli
# or: brew tap ind-igo/cx && brew install cx
# or: curl -sL https://raw.githubusercontent.com/ind-igo/cx/master/install.sh | sh

# One-time grammars for the languages in the repo
cx lang add rust typescript python

# Emit machine-readable instructions for your agent (append to AGENTS.md, CLAUDE.md, etc.)
cx skill >> AGENTS.md

# Cheap-first navigation (TOON by default; add --json when you need JSON)
cx overview src/main.rs
cx symbols --kind fn
cx definition --name resolve_root
cx references --name Symbol
```

The `cx skill` output includes the command reference and an escalation ladder (overview → symbols → definition / references → read full file).

## What It Does

cx is a **local, no-daemon** semantic code navigator: each CLI invocation runs the binary, tree-sitter parses source files, and a **persistent on-disk index** is updated incrementally per project. It serves **overview**, **symbol search**, **definition snippets**, and **references** through one command surface — no language server, no compile step for indexing. References are computed with AST walks so they stay fresh relative to the indexed symbols. Use `cx cache path` to see where the index lives for your install.

Output defaults to [TOON](https://toonformat.dev) for token-efficient structured data; every command supports `--json`.

## Why It Is Agent-Native

| Criterion | Evidence |
|---|---|
| **Agent-first positioning** | GitHub description and README title: *"Semantic code navigation for **AI agents**"* — [repo](https://github.com/ind-igo/cx). | **Agent-specific primitive** | **Token-cost navigation ladder** — README documents deliberately small outputs (`cx overview` ~200 tokens vs full-file reads) and session metrics (*"58% fewer Read calls"* / *"40-55% fewer tokens"*). **TOON-first CLI** tuned for LLM context. These primitives optimize *agent read patterns*, not human editor UX |
| **Autonomy-compatible control plane** | After one-time `cx lang add …` for detected languages, the agent runs `cx …` commands without per-query human confirmation; `.gitignore` and `.cx-ignore` provide policy-like scoping |
| **M2M integration surface** | Rust CLI (`cx-cli` on crates.io), subprocess invocation, `--json` on all commands, TOON output for structured parsing |
| **Identity / delegation** | Runs as the **local OS user** and workspace; per-project index location is reported by `cx cache path` (upstream [README](https://raw.githubusercontent.com/ind-igo/cx/master/README.md) documents `~/.cache/cx/indexes/` keyed by path hash; some package readmes mention a project-root `.cx-index.db`). No cloud agent accounts — attribution is the same as any local dev tool: shell user, VCS, and host audit trails |

## Primary Primitives

| Primitive | Description |
|---|---|
| **`cx overview`** | File table of contents — symbols with kinds and signatures |
| **`cx symbols`** | Project-wide symbol search (`--kind`, `--name`, `--file`) |
| **`cx definition`** | Named symbol body snippet without reading the whole file (`--from`, `--max-lines`) |
| **`cx references`** | Usages with tree-sitter parent context; on-the-fly AST walk |
| **`cx skill`** | Prints agent-facing instructions and command reference |
| **`cx lang`** | Add/list/remove tree-sitter grammars (via language pack) |
| **Incremental index** | First run indexes; later runs update only changed files |
| **Structured output** | TOON default; `--json` optional |

## Autonomy Model

```
Agent (or human) installs cx and runs cx lang add for project languages
    ↓
Agent appends cx skill output to AGENTS.md / CLAUDE.md (optional but recommended)
    ↓
Agent navigates with cx overview / symbols / definition / references
    ↓
cx updates on-disk index incrementally; returns TOON or JSON on stdout
    ↓
Agent escalates to full file read only when a snippet is insufficient
```

---

## Identity and Delegation Model

- **Scope:** All parsing and indexes are **local**; no network API for navigation (grammar downloads for the language pack are a separate one-time fetch).
- **Project binding:** Index keyed by resolved project root (walks up to `.git`; overridable with `--root`).
- **User identity:** Commands run as the invoking **OS user** with that user’s filesystem permissions.

## Human-in-the-Loop Support

Initial setup may require a human (or bootstrap agent) to install the binary and run `cx lang add` for the languages in the repository. Day-to-day navigation does not require per-query human input.

---

## Why Generic Alternatives Do Not Qualify

| Alternative | Why It Fails |
|---|---|
| **LSP / language servers** | Long-lived processes, heavy RAM, per-language editor configuration, often depends on build/index readiness — poor fit for ephemeral agent sessions that only need structured navigation |
| **ripgrep / text search** | Text match, not semantic symbol definition/reference with AST-backed kinds |
| **ctags** | Regex-oriented tags; cx uses tree-sitter queries, persistent incremental index, and agent-oriented output formats (TOON / JSON) |
| **Raw file reads** | Highest token cost; cx is explicitly designed to reduce full-file reads in agent sessions |

## Use Cases

- **Coding agents in large repos** — replace long read chains with `overview` / `definition`
- **CI or sandboxes without LSP** — semantic navigation without compiling the world
- **Multi-language workspaces** — add grammars on demand via `cx lang add`
